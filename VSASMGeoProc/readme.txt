Here is the folder structure:
===============================

1. VC++ Projects and files
----------------------------------
GeoProc: VC++ project to create .\Debug\GeoProc.lib 
GeoProcTest: VC++ project to create .\Debug\GeoProcTest.exe
LASGeoProc: VC++ project to create .\Debug\LASGeoProc.exe
VSASMGeoProc.sln: solution file
Debug: debug folder to include library and executable files after each re-build in VC++

2. MASM Command Line Project:
----------------------------------
ASMGeoProc: under same parent folder with other 3 VC++ projects
ASMGeoProc\src: library source files
ASMGeoProc\include: library include files
ASMGeoProc\lib: static libary GeoProc.lib
ASMGeoProc\ccm.bat: DOS batch file to create GeoProc.lib, GeoProcTest.exe and LASGeoProc.exe
ASMGeoProc\vcDevEnt.bat: pre-run DOS batch file to set up VC++ command line development environment for running ccm.bat
ASMGeoProc\vcvars32.bat: original complete VC++ command line development environment set up DOS batch file.
ASMGeoProc\GeoProcTest.log: a output log file generated by command: GeoProcTest.exe > GeoProcTest.log

3. Data IO
------------------------
LASInput: input LAS file cube.las 
LASOutput: output LAS inside record file cube_inside.las and coordinates file cube_inside.txt
The 2 folders are consumed and shared by both VC++ projects and MASM Command Line project

4. Document
-----------------------------------
ListDesign: inculde List design documents

5. VisualStudio ASM Setup
-----------------------------------
VisualStudio ASM Setup: documents and a how to file to guide to set up MASM development in VC++ enviroment
